# MuseTalk Real-Time Optimized (T2S Pipeline Edition)

Эта версия MuseTalk оптимизирована для работы в реальном времени в связке с моделями Text-to-Speech (T2S). 

## Основные изменения:
- **⚡ Низкая задержка:** Batch size уменьшен до 4, добавлен `torch.compile` для UNet.
- **🚀 GPU Blending (Ultra Fast):** Смешивание лица и фона происходит полностью в видеопамяти (PyTorch Tensors). Нулевые задержки на копирование данных.
- **🧊 VRAM Pre-caching:** Все кадры и маски кэшируются в GPU один раз при инициализации.
- **🧵 Async saving:** Запись PNG-кадров выполняется в отдельном пуле потоков.
- **🎧 Streaming Audio:** Замена `librosa` на `soundfile` для мгновенной загрузки WAV.
- **🖼️ PNG Sequence:** Прямой вывод кадров в папку без лишней перекодировки в видео.
- **🔍 Watcher Mode:** Автоматический мониторинг папки на предмет новых аудио-файлов.

## Установка
1. Убедитесь, что установлены зависимости:
   ```bash
   pip install soundfile torch torchvision opencv-python gradio transformers
   ```
2. Скачайте веса моделей (в папку `./models/`).

## Запуск
Запустите веб-интерфейс:
```bash
python app_realtime.py
```

## Режимы работы

### 1. Manual Control (Ручной режим)
- Выберите видео (аватар) и аудио-файл.
- Нажмите **START INFERENCE**.
- Вы увидите живой поток и кадры будут сохранены в `results/v15/avatars/[ID]/sequence/`.

### 2. Watcher Mode (Автоматический)
Этот режим идеален для связки с T2S моделями:
1. Выберите аватара во вкладке **Watcher Mode**.
2. Укажите папку для мониторинга (например, куда ваша T2S сохраняет результат).
3. Нажмите **START WATCHER**.
4. MuseTalk будет автоматически обрабатывать каждый новый `.wav` файл, как только он появится.

## Требования
- NVIDIA GPU (тестировалось на RTX 4080).
- Windows/Linux.
- Python 3.10+.
